# 素材宫格、Instagram 轮播与文章保真设计

## 目标

统一 Eagle 与 Bear 的素材选择体验，支持在宫格中多选素材；为 Eagle 增加 Instagram 整帖轮播媒体提取；修复 Bear 文章正文缺失；增强 Obsidian 文章收藏，使正文图片本地化并保持图文结构。

## 范围

### Eagle

- 素材候选统一以三列宫格展示。
- 图片、视频、截图等候选可逐项勾选，支持全选和清空。
- Instagram 帖子读取核心轮播区域内的全部图片和视频，不局限于当前可见项。
- 每个已选素材作为独立 Eagle 项目保存。
- 视频缩略图显示播放标识和可获得的时长信息。
- 单项导入失败不阻断其他项目；完成后显示成功数与失败数。

### Bear

- 现有素材候选复用与 Eagle 相同的三列宫格和多选交互。
- 所有已选素材作为附件插入同一条 Bear 笔记，不拆分笔记。
- 保留文章标题、来源链接、描述和正文。
- 修复正文提取不足时只生成标题的问题，重点覆盖 Pangram Pangram Journal 页面。

### Obsidian

- 保持自动收藏逻辑，不显示素材勾选宫格。
- 从文章主体生成 Markdown，保留标题层级、段落、强调、列表、引用、链接、代码块、图片及其文档顺序。
- 正文图片下载到当前 Obsidian 仓库的附件目录，并将 Markdown 图片地址改为仓库内相对路径。
- 图片下载失败时保留原始远程地址，继续生成笔记。
- 不承诺还原网页 CSS、字体和响应式布局；目标是保持 Markdown 可表达的内容结构和图文顺序。

## 非目标

- 不改变 Obsidian 的候选选择界面或确认逻辑。
- 不做网页 CSS 的像素级复制。
- 不抓取 Instagram 帖子核心轮播之外的推荐内容、头像、评论图片或页面装饰。
- 不因本功能重构无关的渠道配置和路由逻辑。

## 交互设计

Eagle 与 Bear 的素材区域使用三列正方形缩略图宫格。每项右上角有勾选控件；选中项同时显示明显边框，避免只依赖颜色。视频项左下角显示播放标识与时长。区域顶部显示“已读取 N 项 · 已选 M 项”以及全选或清空操作。渠道确认按钮展示将处理的素材数量。

宫格组件只负责候选展示、选择状态和提交选中 ID。Eagle 与 Bear 分别解释这些 ID 并执行各自的保存规则。Obsidian 不使用该组件。

## 数据流

### 来源 URL 规范化

页面进入预览前生成规范化来源 URL。清理明确用于来源归因或广告追踪的查询参数，包括 `ref`、`utm_*`、`fbclid` 和 `gclid`，并保留分页、搜索、语言、资源 ID 等可能改变页面内容的功能参数。Eagle、Bear 与 Obsidian 保存的来源链接统一使用规范化 URL；实际页面采集仍可使用浏览器当前 URL，避免清理参数影响已打开页面。

### 页面采集

扩展采集页面元数据、正文结构和候选素材。对于 Instagram 帖子，采集器限定在主帖轮播容器内，读取轮播项顺序、媒体类型、图片地址、视频地址、封面、时长和轮播序号。若 Instagram 只在交互后挂载部分轮播项，采集器应在不跳出当前帖子的前提下遍历轮播，并去重恢复原顺序。

### 预览与选择

路由预览返回稳定候选 ID、类型、预览地址、标签和默认选择状态。弹窗通过共享宫格渲染 Eagle 与 Bear 候选，提交选中 ID 数组。服务端只处理仍存在于预览候选集合中的 ID，避免接受任意资源地址。

### 保存

- Eagle 逐项导入已选候选，并聚合每项结果。
- Bear 创建一条笔记，把已选附件按候选顺序插入笔记。
- Obsidian 将正文 HTML 转换为结构化 Markdown；在转换过程中下载正文图片，写入附件目录，并替换为相对链接。

## 正文提取策略

正文提取按以下优先级选择内容：语义化 `article` 或 `main` 中的文章节点、页面显式正文容器、结构化数据中的 article body，最后才使用描述元数据。需要排除导航、侧栏、页脚、推荐卡片和重复标题。

Bear 与 Obsidian 共用正文识别结果，但输出不同：Bear 使用其支持的富文本或 Markdown 导入格式并附加所选素材；Obsidian 生成 Markdown 并本地化正文图片。若正文仍为空，至少写入非空摘要和来源链接，而不是只留下标题。

## Instagram 媒体策略

优先使用页面已解析的具体媒体 URL，以准确对应轮播项。视频 URL 无法直接下载或已经过期时，使用帖子 URL 与轮播序号执行后端回退提取。候选按轮播顺序去重；图片与视频可以混合多选。回退仍无法获得目标媒体时，记录该项失败，不以错误媒体替代。

## 错误处理

- 页面采集失败时保留现有基础页面候选，不伪造轮播媒体。
- Eagle 和 Bear 的单项失败隔离处理，并返回成功数、失败数及简短原因。
- Bear 附件部分失败时仍创建含正文和成功附件的单一笔记。
- Obsidian 图片下载失败时回退远程 URL；正文生成不因单图失败而中断。
- 临时下载文件在导入完成或失败后清理。

## 测试与验收

自动化测试覆盖：

- 共享宫格能渲染混合素材、切换多选、全选、清空并提交稳定 ID。
- Eagle 和 Bear 使用同一选择交互，但执行不同保存语义。
- Instagram 普通视频帖和图文视频混合轮播能按顺序产生全部核心媒体候选。
- Instagram 重复挂载或懒加载不会生成重复候选。
- Eagle 多选逐项保存，部分失败时继续并正确汇总。
- Bear 多选附件进入同一条笔记并保持候选顺序。
- Pangram Pangram Arrows 页面能提取非空正文或有效摘要，不再只有标题。
- Obsidian 保持标题、段落、列表、链接和图片顺序，图片写入仓库并使用相对路径。
- Obsidian 单图下载失败时保留远程链接。
- 来源 URL 清理已知追踪参数，但保留未知参数和功能参数；例如 `https://pangrampangram.com/blogs/journal/arrows?ref=sidebar` 保存为 `https://pangrampangram.com/blogs/journal/arrows`。

手动验收使用用户提供的 Instagram 与 Pangram Pangram URL，验证弹窗宫格、实际多选保存结果、Bear 单笔记附件，以及 Obsidian 中的本地图文笔记。

## 兼容与迁移

现有单选任务继续可用：服务端将旧的单个候选 ID 规范化为单元素数组。没有素材候选的页面保持当前预览。现有 Obsidian 路径和笔记命名规则不变，仅增加文章正文图片附件。
